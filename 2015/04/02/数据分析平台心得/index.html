开发环境

服务端所使用的包均在package.json中注册,前端所使用的均在bower.json中注册
其中主要运用的技术如下:

  • Node.js

    Node.js是一个基于Chrome JavaScript运行时建立的平台， 用于方便地搭建响应速度快、易于扩展的网络应用。Node.js 使用事件驱动， 非阻塞I/O 模型而得以轻量和高效，非常适合在分布式设备上运行的数据密集型的实时应用。Node.js中集成了包管理器npm

  • Express.js

    express 是一个基于Node.js的轻量级WEB服务器框架,本项目使用版本为3.4.0. 注意!在此版本中express已集成诸如session static logger cookieParser 等功能,而在4.0版本中,大部分功能已剥离并需以中间件的方式手动安装.

  • Yeoman

  1. Yo

    Yo 是一个 Web 应用的架构（scaffolding）工具。它提供了非常多的模板，用来生成不同类型的 Web 应用。这些模板称为生成器（generator）。社区也贡献了非常多的生成器，适应于各种不同的场景。通过 Yo 生成的应用使用 Grunt 来进行构建，使用 Bower 进行依赖管理。

  2. Grunt.js

    自动化。对于需要反复重复的任务，例如压缩（minification）、编译、单元测试、linting等，自动化工具可以减轻你的劳动，简化你的工作。当你正确配置好了任务，任务运行器就会自动帮你或你的小组完成大部分无聊的工作。

  3. Bower.js

    Bower 是 twitter 推出的一款包管理工具，基于nodejs的模块化思想，把功能分散到各个模块中，让模块和模块之间存在联系，通过 Bower 来管理模块间的这种联系。

  • Angular.js

    AngularJS是Google开发的纯客户端JavaScript技术的WEB框架,用于扩展、增强HTML功能,它专为构建强大的WEB应用而设计。最为核心的是：MVVM、模块化、自动化双向数据绑定、语义化标签、依赖注入，等等。

  • Require.js

    RequireJS的目标是鼓励代码的模块化，它使用了不同于传统<script>标签的脚本加载步骤。可以用它来加速、优化代码，但其主要目的还是为了代码的模块化。它鼓励在使用脚本时以module ID替代URL地址。

  • jQuery

    轻量级的js库,作为众多插件的基础类库

项目启动

  1. 以下整个前端开发流程以node.js为基础,从官网安装http://www.nodejs.org/ Node.js.
  2. 安装ruby是为了实现对sass的支持，下载Rubyinstaller http://rubyinstaller.org/ 进行安装，安装完成之后输入如下命令安装sass和compass:
    gem install sass gem install compass
    成功之后可在命令行检查是否安装好：
    sass –v compass –v
  3. 安装grunt　　
    npm install -g grunt-cli
    使用方法：http://www.gruntjs.org/
  4. 安装bower
    npm install –g bower
    使用方法：http://bower.io/
  5. 安装 node依赖包
    npm install
  6. 安装bower依赖包
    bower install
  7. 安装并启动mongodb
    下载地址:https://www.mongodb.org/
    mongod --dbpath=d:/db
  8. 启动项目
    grunt server

项目结构

主要结构

  • server 文件夹为后台服务器主要目录
  • app 文件夹为前端文件夹,由express作为静态资源服务器

入口文件为 server.js

服务端结构

server/main.js 主文件

引入功能中间件,并进行路由配置
其中静态文件服务器配置为 app文件夹

app.use(express.static(path.join(__dirname, '../app')));

注意app.use()引入顺序,因静态服务器引用位于路由之前,所以如果在静态服务器中存在的文件路径无法通过之后引入的路由匹配

通过 mongoose 连接mongodb

server/config

config.js 配置文件
主要包含一些基本的配置信息

server/logs 日志文件夹

基本服务器日志信息

  • httpProxy.js http请求简单实现，负责连接远程服务器获取数据
  • logger.js 日志处理
  • mongoModal.js mongoose model 建立
  • response.js 对res方法进行封装,使得在使用res时设置表头'Content-Type', 'application/json; charset=UTF-8',并输出日志

app/script/routers

本项目前端路由文件为angular-ui-router

  • 在应用程序的整个用户界面和导航中，一个状态对应于一个页面位置
  • 通过定义controller、template和view等属性，来定义指定位置的用户界面和界面行为
  • 通过嵌套的方式来解决页面中的一些重复出现的部位
  • 在route中以.申明继承状态

状态被激活时，它的模板会自动插入到父状态对应的模板中包含ui-view属性的元素内部。如果是顶层的状态，那么它的父模板就是index.html。

有三种方法来激活状态：

  1. 调用$state.go()方法，这是一个高级的便利方法；
  2. 点击包含ui-sref指令的链接；
  3. 导航到与状态相关联的 url。

协同工作

编码规范

  1. 谷歌javascrpit代码风格
  2. Angular.js规范
  • 不要直接操作DOM。
  • View中包含尽量少的逻辑。
  • 在controller和service中不能出现html的DOM和CSS代码。
  • controller中公用的逻辑推向service（factory，value，config)，采用IOC的注入，提高代码的重用度，修改的单一点，开闭原则。
  • controller应该只包含业务逻辑，对于数据模型的格式化过滤尽量交给angular框架filter等处理。
  • 对controller之间如果不是强依赖，只是弱引用则最好用事件$emit,$on,$broadcast。
  • 不要在应用中使用全局定义的控制器。
  • 必须完整声明依赖。
  • 需要使用jquery插件时，将jquery插件封装成dirctive。
  1. 命名规范
  • 一般情况下采用驼峰模式
  • 定义controller，directive，filter，service时候应保证与模块名有一定的关联性，在各处引入时所取的变量的命名含义相同,如:
define([
    'underscore',
    'routers/routers',
    'common/appControllers',
    'worldMap/controllers'
], function(_, routers, ac, wc) {

    var controllers = {};
    controllers = _.extend(controllers, ac, wc);
